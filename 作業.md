# EchoMate 開発ロードマップ（Claude Code 連携用）

# Phase 1: コア機能の実装と近代化（完了済み）

以下の機能および品質向上はすべて実装・検証済みです。正常動作を確認しました。

### 1. 基本機能・UX
- **会話の動的付与・フィラー先行再生**: 自然な対話と待ち時間の解消。
- **再生キャンセル (Barge-in)**: プレイヤー発信時の AI 中断。
- **記憶・成長システム**: 親密度、プレイスタイル反映、エピソード記憶。
- **Webソース学習**: URL からの知識抽出と RAG 統合。
- **UI フィードバック**: 「AI 思考中...」インジケーター、メンタルグラフ、プロファイル表示。
- **マルチモニター対応**: Vision 解析対象モニターの選択機能。

### 2. システム近代化・堅牢化
- **通信・ライブラリ刷新**: 全通信の `httpx` 化、`pyaudiowpatch` による音声安定化。
- **データの安全性**: `tempfile + os.replace` によるアトミックな JSON 保存。
- **履歴の SQLite 移行**: 肥大化するログ・記憶を `PatronDB` (SQLite) へ移行。
- **AI 応答の正常化**: 制約注入による定型文落下問題の解消。
- **笑い声フィルタ**: 正規表現 `[あはひふへほっ]+` による笑い声・相槌の自動無視。
- **GPU (CUDA) 加速の有効化**: `voice.py` の `WhisperModel` にて CUDA 対応済み。

# Phase 2: 最強設定 (RTX 4070 Ti SUPER) と AI クオリティ向上（未着手）

## 【1】音声認識 (STT) の高精度化 (LoL 最適化)
**目的**: RTX 4070 Ti SUPER のパワーを活かし、LoL 専門用語を完璧に聞き取ります。

- **モデル昇格 (未設定)**: `.env` および `voice.py` で `model_size` を `medium` に引き上げ。
- **LoL 辞書 (Initial Prompt)**: `voice.py` の辞書に「チャンピオン, ミッド, ガンク, フラッシュ, インベード, バロン, ドラゴン, リコール, ウルト, テレポート, スマイト」等を登録。
- **VAD 調整**: `transcribe` 時の `vad_parameters` で `min_speech_duration_ms=100` 等を設定し、小さな声も拾えるよう調整。

## 【2】AI 感情分析 (Sentiment) による共感力の向上
**目的**: プレイヤーの「イライラ」「喜び」を言葉から察し、AI が寄り添えるようにします。

- **sentiment_analyzer.py**: 作成済み。
- **ai.py / main.py 連携**: 判定結果を AI のプロンプトに注入。「ユーザーは現在 [感情] な状態です」と伝え、応答トーンを変化させる（要確認）。

## 【3】システム堅牢化（文字化け・JSON エラー対策）
**目的**: ログの `ぁE`（文字化け）を撲滅し、分析エラーをゼロにします。

- **ログエンコード修正**: `main.py` の `_setup_logging` で `encoding="utf-8"`, `errors="replace"` を徹底。
- **分析プロンプト強化**: `patron_analyzer.py` の指示を強化。「JSON 以外は一切喋るな」と厳命。
- **パースロジック強化**: 正規表現で `{` から **最後** の `}` までを確実に抽出。

## 【4】UI/UX: ゲーム音キャプチャの可視化 [NEW]
**目的**: ゲーム音が正しくキャプチャされているか GUI で確認できるようにします。

- **ゲーム音インジケーター**: `gui.py` の「マイク設定」タブに、ゲーム音（ループバック）の入力レベルを表示するプログレスバーを追加。
- **AudioMonitor クラス**: `MicMonitor` と同様の仕組みで、ループバックデバイスの RMS レベルを取得・公開するクラスを実装。

## 【5】音声の重複再生防止 [NEW]
**目的**: 同一イベントが短時間に重なった際の「連呼」を防止します。

- **デバウンス処理**: `main.py` の `_handle_game_event` にて、同一イベント（例: kill）が 3秒以内に再発した場合は発話をスキップする。
