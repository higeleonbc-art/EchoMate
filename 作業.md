# EchoMate 追加機能仕様書（Claude Code向け）

## 【実装完了】これまでのタスク（1〜7）
以下の機能はすべて実装・検証済みです。

1.  **質問・掘り下げの動的付与**: 会話の文脈に応じた自発的な問いかけ。
2.  **フィラー（相槌）の先行再生**: LLM生成を待たずに「えーっと」「なるほど」等を再生。
3.  **再生キャンセル（Barge-in）**: プレイヤーの発言時にAIの音声を中断。
4.  **過去の話題（recent_topics）の掘り起こし**: RAGによる話題の再利用。
5.  **感情別 VOICEVOX スタイル切り替え**: テンションに連動した声色変化。
6.  **Webソース型学習機能**: プレイヤー提供のURLからゲーム知識をスクレイピング・要約。
7.  **「良き隣人」成長システム**:
    *   親密度（Bond Level）の動的蓄積とGUI表示。
    *   プレイスタイル分析結果のAI人格（トーン）への反映。
    *   エピソード記憶の優先抽出。
    *   マルチスレッドセーフな I/O マージロジック（Safe Save）。

## 【改善建議タスク】システムの堅牢化と品質向上
プロジェクト・クオリティ・レビュー（2026-04-16）に基づいた、次期フェーズでの実施を推奨する改善案です。

### 1. データの永続化と安全性（最優先）
*   **アトミック書き込みの導入**: `user_profile.py` の `save()` メソッドを、一時ファイルへの書き出し → リネーム（`os.replace`）方式に変更し、書き込み中のフリーズによる JSON 破損を完全に防ぐ。
*   **履歴データの SQLite 移行**: `user_profile.json` 内の `memorable_episodes` や `growth_observations` は増大し続けるため、これらを `PatronDB` (SQLite) へ移行し、メモリ負荷と I/O 効率を改善する。

### 2. コンピュータビジョン（CV）の信頼性向上
*   **時系列フィルタのプリセット適用**: 誤検知が目立つ領域（キルフィード等）に対し、`cv_config.json` 内で `min_hits: 2` などを標準設定とし、瞬間的なノイズをカットする。
*   **オンデマンド Vision 解析のフォールバック**: オーディオトリガーによる VLM 呼び出しがタイムアウトやエラーになった際の例外処理を強化し、AI が「沈黙」せず別の話題へ遷移するようにする。

### 3. UX・フィードバックの強化
*   **思考ステータスの可視化**: LLM や VLM の呼び出し中に GUI 上で「AI 思考中...」といったインジケーターを表示し、フリーズしていないことをユーザーに伝える。
*   **マルチモニター対応**: `VisionAnalyzer` がプライマリモニター以外もキャプチャできるようにモニター選択機能を GUI に実装する。
