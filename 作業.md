# 現在のシステム構成（2026-04-18 時点）

## プロンプト優先順位（get_response）
```
1. 会話力学（Layer 1+2 or Layer 2のみ）
2. キャラクター人格（system_prompt）
3. 長期記憶キャッシュ（ai_memory.db）
4. ルール・制約
5. 常時ルール（最新発言反応・繰り返し禁止）
6. RAG（user_profile / game_knowledge / 画面状況）
7. トーン補正（ゴリ押し/慎重）
8. 感情トーン・観察メモ
9. セッション内過去話題
10. ゲーム状況・記憶
11. 繰り返し防止（直前5発言）
```

## モデル別 dynamics 注入ルール
| モデル | get_response | _call_with_validation(通常) | _call_with_validation(lightweight) |
|--------|-------------|---------------------------|-------------------------------------|
| echomate-* | Layer 2のみ | Layer 2のみ | Layer 2のみ |
| その他(qwen3等) | Layer 1+2 | Layer 1+2 | Layer 2のみ |

## リアクション生成方式
- ゲームイベント時: **60%テンプレート（即時）/ 40%LLM（自然応答）** のランダム混合
- プレイヤー発言応答: 常にLLM（/api/chatエンドポイント）

# Phase 9: 次の作業候補

## 9.1 echomate-* キャラ別 Modelfile に Layer 2 を焼き込む【優先度: 中】

現状: echomate-base には Layer 1 のみ焼き込み。Layer 2（関係性モディファイア）は毎回動的注入。

検討事項:
- `acquaintance` ステージをデフォルトとして焼き込むことで初期応答が改善できる
- ただし bond_level が変わるたびに Modelfile を再作成するのは現実的でない
- → Layer 2 は引き続き動的注入が最適（現状維持でOK）

## 9.2 Modelfile 整備（キャラクター別）【優先度: 中】

`presets/base.Modelfile` から各キャラのModelfileを派生させる計画:
```
echomate-echo     — エコー（現在: characters.json で管理）
echomate-michiko  — ミチコ
echomate-kid      — キッド
```

各 Modelfile は `FROM echomate-base` を基底にキャラ人格を焼き込む。
→ ai.py の sys_parts からキャラ system_prompt 注入が不要になりトークン節約

## 9.3 本格ファインチューニング（将来: Unsloth）【優先度: 低】

前提条件: `patron_logs.db` + `ai_memory.db` に 2000+ 会話ペアが蓄積されてから着手。
ツール: [Unsloth](https://github.com/unslothai/unsloth)（無料・LoRA/QLoRA）

手順:
1. DB から会話ペア抽出 → ShareGPT 形式に変換
2. Unsloth で qwen3:8b をファインチューニング
3. `.gguf` でエクスポート → `ollama create echomate-finetuned -f Modelfile`

## 9.4 性能監視・その他【優先度: 低】
- **LLM応答時間の可視化**: GUI に応答速度インジケーター追加
- **自動クリップ作成**: 会話が盛り上がった場面を動画保存
- **マルチモーダル強化**: 画面の特定箇所を高精度に読み取る精度向上

# 既知の制約・設計上の注意点

- `game_knowledge.json` は `patron_analyzer.py` の `_extract_curiosities()` で自動更新。手動編集は非推奨。
- `user_profile.json` と `patron_logs.db` の二重管理が残存（SQLite移行中）。`user_profile.py` の `patron_db` 連携で最終的にSQLite一本化予定。
- `ai_temp_memory.db` はセッションごとにクリアされない（`compress_and_save()` でアーカイブ後も残る）。容量に注意。
- `echomate-base` モデルを事前にビルドしていない場合 `.env` の `LLM_MODEL` を `qwen3:8b` に変更して起動すること。
